# 页面结构

github (opens new window) @ szm8991

HTML 标签都有具体语义，虽然技术上可以使用 div 标签表示大部分内容，但选择清晰的语义标签更容易让人看明白。比如 h1 表示标题、p 表示段落、em 表示强调内容。

HTML 标签可以互相嵌套包裹，通过标签间的嵌套包裹绘制页面结构。

# 通用容器

通用容器使用div来定义，通用容器标签出现较早，是使用最多的 THML 标签，用于定义没有语义的容器。

<div>
  div内容
</div>

# 标题

标题使用h1 ~ h6来定义，用于突出显示文档内容主题。

  • 从 h1 到 h6 对搜索引擎来说权重会越来越小
  • 页面中最好只有一个 h1 标签
  • 标题最好不要嵌套如 h1 内部包含 h2
<h1>标题内容</h1>
<h2>标题内容</h2>
<h3>标题内容</h3>
<h4>标题内容</h4>
<h5>标题内容</h5>
<h6>标题内容</h6>

# 页眉

页眉使用header来定义，用于显示文档页眉内容。

<header>
  <div>
    header内容
  </div>
</header>

# 页脚

页眉使用footer来定义，用于显示文档页脚内容。

<footer>
  <div>
    footer内容
  </div>
</footer>

# 导航栏

导航栏使用nav来定义，用于显示文档导航栏内容。

<nav>
  <div>
    nav1
  </div>
  <div>
    nav2
  </div>
</nav>

# 主要区域

主要区域使用main来定义，用于显示文档主要区域内容。

  • 一个页面中 main 元素最好只出现一次
<main>
  <div>
    main内容
  </div>
</main>

# 内容区域

内容区域使用article来定义，用于显示独立的自包含内容区域。

<main>
  <article>
    <div>
      article内容
    </div>
  </article>
</main>

# 内容区块

内容区块使用section来定义，一般用于显示一组相似内容的排列组合。

<main>
  <article>
    <section>
      <div>区块内容1</div>
    </section>
    <section>
      <div>区块内容2</div>
    </section>
  </article>
</main>

# 侧边栏

侧边栏使用aside来定义，用于显示与主要区域无关的内容。

<body>
  <main>
    <article></article>
  </main>
  <aside>
    <div>侧边栏内容</div>
  </aside>
</body>

# 文本相关

# 段落

p 标签表示一个段落内容。

<p>Enjoy when you can, and endure when you must.</p>

# 原样输出

pre 标签会原样显示标签内包裹的内容(包括换行符、空白)

<pre>
    这是pre标签的显示效果
            这是换行后的内容，空白和换行都会保留
</pre>

# 换行符

br 标签表示换行符(自闭合标签)

<br />

# 水平分割线

hr 标签表示水平分割线(自闭合标签)

<hr />

# 通用文本

span 标签表示通用文本，一般用于表示语义不清晰的文本。

<span>这是一个span文本</span>

# 时间标签

time 标签用于表示日期或时间

<time>2020-08-20</time>

# 缩写内容

abbr 标签用于表示缩写内容

# 表格

表格在网页开发中使用频率非常高，尤其是数据展示时。

# 基本使用

标签 说明
caption 表格标题
thead 表头部分
tbody 表格主体部分
tfoot 表格尾部
<table border="1">
  <caption>
    表格标题
  </caption>
  <thead>
    <tr>
      <th>表头1</th>
      <th>表头2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>内容1</td>
      <td>内容2</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>总计1</td>
      <td>总计2</td>
    </tr>
  </tfoot>
</table>
